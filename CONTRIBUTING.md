# Contributing to awesome-dcs-world

Thanks for wanting to add something. Keep it short, keep it useful.

---

## Format d'une entrée

```
- [Nom](https://url.com) — Description courte de ce que ça fait.
```

- Le tiret de séparation est un tiret cadratin `—` (pas un tiret simple `-`)
- Description en français ou en anglais, au choix, mais pas les deux pour la même entrée
- **Maximum 120 caractères** pour la description, espaces compris
- Pas de point final

---

## Badges

Ajoute un badge **après le lien**, avant le `—`, si c'est pertinent :

| Badge | Quand l'utiliser |
|-------|-----------------|
| `[inactive]` | Plus maintenu depuis > 1 an, mais toujours utile |
| `[paid]` | Payant ou freemium avec fonctions essentielles payantes |
| `[WIP]` | En développement actif, instable ou incomplet |

Exemple : `- [MonMod](https://url.com) [WIP] — Un mod pour les F-16 en cours de dev.`

---

## Règles de qualité

- **Liens vérifiés** : teste le lien avant de soumettre. Un lien mort = PR refusée.
- **Ordre alphabétique** dans chaque section. Si tu ne sais pas où mettre ton entrée, mets-la dans l'ordre logique et indique-le dans la PR.
- **Pas de self-promo** sans usage communautaire prouvé. Si tu as créé l'outil toi-même, montre qu'il est utilisé (stars GitHub, thread sur les forums, etc.).
- **Une entrée par ressource**. Pas de doublons, même si l'URL a changé.
- **Pas d'outils exclusivement commerciaux** sans alternative libre ou sans valeur ajoutée claire pour la communauté.

---

## Ce qu'on refuse

- Liens morts ou redirigés vers une page générique
- Outils sans utilisateurs visibles (0 stars, 0 posts, projet sorti la semaine dernière)
- Doublons (vérifie avec `Ctrl+F` avant de soumettre)
- Outils purement commerciaux dont le seul objectif est de vendre quelque chose
- Descriptions vagues du type "un outil DCS" ou "utile pour les missions"
- Tout ce qui ne fonctionne pas avec la version stable actuelle de DCS, sans mention explicite

---

## Soumettre une PR

1. **Fork** ce repo
2. Crée une branche avec un nom parlant : `add-tacview-plugin` ou `fix-dead-link-srs`
3. Fais ta modification dans `README.md`
4. Ouvre une **Pull Request** vers `main`
5. Remplis le template de PR — pas besoin de roman, juste les cases à cocher et une ligne d'explication

Le mainteneur passe en revue les PR régulièrement. Si ta PR reste sans réponse plus de 2 semaines, ping dans les commentaires.

---

## Questions ?

Ouvre une issue. Pas besoin de cérémonie.
