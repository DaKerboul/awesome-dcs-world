# Contributing to awesome-dcs-world

Thanks for wanting to add something. Keep it short, keep it useful.

---

## Entry format

```
- [Name](https://url.com) — Short description of what it does.
```

- The separator is an em dash `—` (not a regular hyphen `-`)
- Description in English
- **120 characters max** for the description, spaces included
- No trailing period

---

## Badges

Add a badge **after the link**, before the `—`, when relevant:

| Badge | When to use |
|-------|-------------|
| `[inactive]` | No longer maintained for 1+ year, but still useful |
| `[paid]` | Paid or freemium with essential features behind a paywall |
| `[WIP]` | Actively in development, unstable or incomplete |

Example: `- [MyMod](https://url.com) [WIP] — A mod for F-16s, still in development.`

---

## Quality rules

- **Working links**: test the link before submitting. Dead link = PR rejected.
- **Alphabetical order** within each section. If you're unsure where it fits, pick the most logical spot and mention it in the PR.
- **No self-promo** without proven community usage. If you built the tool yourself, show that people use it (GitHub stars, forum thread, etc.).
- **One entry per resource**. No duplicates, even if the URL changed.
- **No purely commercial tools** without a free tier or clear community value.

---

## What we reject

- Dead links or links redirecting to a generic page
- Tools with no visible users (0 stars, 0 posts, project released last week)
- Duplicates (check with `Ctrl+F` before submitting)
- Purely commercial tools whose only purpose is to sell something
- Vague descriptions like "a DCS tool" or "useful for missions"
- Anything that doesn't work with the current stable version of DCS, without an explicit note

---

## Submitting a PR

1. **Fork** this repo
2. Create a branch with a descriptive name: `add-tacview-plugin` or `fix-dead-link-srs`
3. Make your change in `README.md`
4. Open a **Pull Request** against `main`
5. Fill in the PR template — no essay needed, just the checkboxes and one line of context

The maintainer reviews PRs regularly. If your PR sits without a response for more than 2 weeks, ping in the comments.

---

## Questions?

Open an issue. No ceremony needed.
